كيفية أتمتة إنشاء صفحات المنتجات الإلكترونية باستخدام Next.js و ChatGPT API

✍️ UcfZem Tech 📅 يونيو 2026 ⏱️ 12 دقيقة قراءة

المقدمة: كابوس صفحات المنتجات

لديك 200 منتج للنشر عبر الإنترنت. كل صفحة تحتاج عنواناً محسناً للبحث (SEO) ووصفاً جذاباً ومواصفات فنية ووصف meta. بالسرعة اليدوية، هذا يعني 3 أسابيع من العمل المتكرر — و منافسوك đã أطلقوا كتالوجهم بالفعل.

في هذا الدليل، سنبني خط أنفاق أتمتة كامل يحول CSV بسيط من المنتجات إلى صفحات تجارة إلكترونية جاهزة للنشر، من خلال الجمع بين قوة Next.js و ذكاء ChatGPT.

Section 1: خطوات خط الأنفاق التقني

1.1 الهيكل العام

يعتمد حلنا على ثلاث ركائز:

  • ملف CSV منظم يحتوي على البيانات الخام لمنتجاتك
  • مسار API في Next.js ينسق الاتصال بـ OpenAI
  • مكون React يعرض ويدير التوليد في الوقت الفعلي

1.2 مسار API في Next.js

هذا هو قلب نظامنا. إليك نقطة النهاية التي تستقبل بيانات المنتج وترجع المحتوى المولّد:

TypeScript
// app/api/generate-product/route.ts
import { NextRequest, NextResponse } from 'next/server';
import OpenAI from 'openai';

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,
});

interface ProductInput {
  name: string;
  category: string;
  price: number;
  features: string[];
  brand?: string;
}

export async function POST(request: NextRequest) {
  try {
    const product: ProductInput = await request.json();

    const prompt = `
      أنت خبير في كتابة محتوى التجارة الإلكترونية.
      أنشئ صفحة منتج كاملة لـ:
      - الاسم: ${product.name}
      - الفئة: ${product.category}
      - السعر: ${product.price}€
      - العلامة التجارية: ${product.brand || 'غير محددة'}
      - الميزات: ${product.features.join(', ')}

      قدم: عنوان SEO (حد أقصى 60 حرف)، وصف meta
      (حد أقصى 155 حرف)، وصف كامل، و 5 نقاط رئيسية.
      `;

    const completion = await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [{ role: 'user', content: prompt }],
      temperature: 0.7,
      max_tokens: 1500,
    });

    const generatedContent = completion.choices[0].message?.content;

    return NextResponse.json({
      success: true,
      content: generatedContent,
      usage: completion.usage,
    });
  } catch (error) {
    return NextResponse.json(
      { success: false, error: 'خطأ في التوليد' },
      { status: 500 }
    );
  }
}

1.3 مكون الواجهة الأمامية React

مكون نظيف يتعامل مع الإرسال وعرض النتيجة:

TypeScript
// components/ProductGenerator.tsx
'use client';
import { useState } from 'react';

export default function ProductGenerator() {
  const [loading, setLoading] = useState(false);
  const [result, setResult] = useState('');
  const [form, setForm] = useState({
    name: '', category: '',
    price: 0, features: '',
  });

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setLoading(true);

    const res = await fetch('/api/generate-product', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        ...form,
        price: Number(form.price),
        features: form.features.split(',').map(f => f.trim()),
      }),
    });

    const data = await res.json();
    setResult(data.content);
    setLoading(false);
  };

  return (
    <div className="generator-card">
      <h2>مولد صفحات المنتجات</h2>
      <form onSubmit={handleSubmit}>
        <input placeholder="اسم المنتج"
          onChange={e => setForm({...form, name: e.target.value})} />
        <input placeholder="الفئة"
          onChange={e => setForm({...form, category: e.target.value})} />
        <input type="number" placeholder="السعر"
          onChange={e => setForm({...form, price: e.target.value})} />
        <textarea placeholder="الميزات (مفصولة بفواصل)"
          onChange={e => setForm({...form, features: e.target.value})} />
        <button type="submit" disabled={loading}>
          {loading ? 'جارٍ التوليد...' : 'توليد الصفحة'}
        </button>
      </form>
      {result && (
        <div className="result"
          dangerouslySetInnerHTML={{ __html: result }} />
      )}
    </div>
  );
}

1.5 Rate Limiter

احمِ نقطة النهاية الخاصة بك ضد سوء الاستخدام:

TypeScript
// lib/rate-limiter.ts
const requests = new Map<string, number[]>();

export function rateLimit(
  ip: string,
  maxRequests = 50,
  windowMs = 60000
): boolean {
  const now = Date.now();
  const timestamps = requests.get(ip) || [];
  const valid = timestamps.filter(t => now - t < windowMs);

  if (valid.length >= maxRequests) return false;

  valid.push(now);
  requests.set(ip, valid);
  return true;
}

Section 2: الأخطاء التي يجب تجنبها والممارسات الصحيحة

إليك أكثر الأخطاء شيوعاً وكيفية التغلب عليها:

❌ خطأ شائع ✅ الحل 💡 التأثير
إرسال الكتالوج بالكامل دفعة واحدة المعالجة على دفعات من 10-20 منتج يقلل أوقات الانتظار بنسبة 90%
لا تحقق من استجابات الذكاء الاصطناعي تحليل JSON المرتجع والتحقق منه يلزم 100% من أخطاء العرض
مفتاح API مكشوف في العميل استخدام مسارات API حصرياً أمان حرج
لا تقييد المعدل تنفيذ عداد حسب العنوان IP يتجنب فواتير مفاجئة
محتوى مولّد غير مُتحقق منه مراجعة يدوية + فلتر إشراف جودة وامتثال قانوني

الخطأ رقم 1: إرسال النصوص الأولية بدون هيكل. ChatGPT يُرجع نصاً غير منظم. الحل: فرض تنسيق JSON صارم في نصك الأولي وتحليل الاستجابة.

الخطأ رقم 2: تجاهل الرموز المميزة. كل استدعاء لـ GPT-4o يكلف في الرموز. لـ 1000 منتج، يرتفع بسرعة. نصيحة: استخدم GPT-4o-mini للوصف البسيط و GPT-4o فقط للمحتوى الحرج.

الخاتمة: من الأتمتة إلى الهيمنة

أتمتة صفحات المنتجات ليست ترفاً — إنها ميزة تنافسية حيوية. مع Next.js و ChatGPT، لديك خط أنفاق سريع وآمن وقابل للتوسع.

الخطوات الرئيسية للتذكر:

  1. هيكل بيانات منتجاتك في CSV نظيف
  2. بناء مسار API في Next.js متين مع تقييد المعدل
  3. تحقق دائماً من استجابات الذكاء الاصطناعي قبل الإدراج
  4. تنفيذ تخزين مؤقت لتحسين تكاليف API
  5. ابدأ صغيراً، ثم توسع تدريجياً

منافسوك ما زالوا يقضون أيامهم في كتابة صفحات المنتجات يدوياً. أنت يمكنك أتمتة العملية بأكملها وتخصيص وقتك لما يهم حقاً: البيع.
